# Knowledge: Patterns

## Purpose

This directory contains **patterns** — practices, structures, or approaches that have proven effective and are intended for reuse across features or projects.

## Rules

- Patterns must be **explicitly linked** from intents or decisions to be used
- Patterns are **advisory**, not authoritative
- Patterns should reference concrete evidence or outcomes
- Patterns are never auto-included in context bundles

## Structure

Each pattern file should:
- Define what the pattern is
- Explain when to use it
- Provide examples or evidence
- Link to related decisions or features

## Usage

Patterns are included in context bundles only when explicitly referenced by:
- Feature intents
- Decisions
- Other knowledge artifacts

Patterns are not automatically discovered or included.
