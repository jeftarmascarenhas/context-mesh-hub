# Knowledge: Anti-Patterns

## Purpose

This directory contains **anti-patterns** — practices, structures, or approaches that have caused issues, risks, or failures and should be explicitly avoided.

## Rules

- Anti-patterns must be **explicitly linked** from intents or decisions to be used
- Anti-patterns are **advisory**, not authoritative
- Anti-patterns should reference concrete evidence or outcomes
- Anti-patterns are never auto-included in context bundles

## Structure

Each anti-pattern file should:
- Define what the anti-pattern is
- Explain why it's problematic
- Provide examples or evidence
- Link to related decisions or features

## Usage

Anti-patterns are included in context bundles only when explicitly referenced by:
- Feature intents
- Decisions
- Other knowledge artifacts

Anti-patterns are not automatically discovered or included.
